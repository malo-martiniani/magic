# Session Summary - 2026-03-22

## User goal
- Learn XMage codebase and set up local development workflow for custom server play with friends.

## What was mapped
- Root project is a Maven multi-module build: modules listed in root pom.xml.
- Core engine module: Mage.
- Shared/network/common module: Mage.Common.
- Server runtime module: Mage.Server.
- Client runtime module: Mage.Client.
- Card/set implementation module: Mage.Sets (uses nonstandard source layout with `sourceDirectory=src`).
- Extra gameplay/tournament/deck/player plugins: Mage.Server.Plugins and Mage.Plugins.
- Optional admin GUI for server control: Mage.Server.Console.

## Key entry points confirmed
- Server main class: mage.server.Main
  - Source: Mage.Server/src/main/java/mage/server/Main.java
  - Also set in Mage.Server/pom.xml mainClass.
- Client main class: mage.client.MageFrame
  - Source: Mage.Client/src/main/java/mage/client/MageFrame.java
  - Also set in Mage.Client/pom.xml mainClass.
- Admin console main class (optional): mage.server.console.ConsoleFrame
  - Source: Mage.Server.Console/src/main/java/mage/server/console/ConsoleFrame.java

## Card/set location details
- Set registry scans package `mage.sets` from class `mage.cards.Sets`.
- Set definitions are in Mage.Sets/src/mage/sets/*.java (classes extend ExpansionSet).
- Individual card classes are in Mage.Sets/src/mage/cards/<letter>/*.java (classes extend CardImpl).

## Build/run guidance prepared
- Build all modules from repo root with Maven (`mvn clean install -DskipTests`).
- For first pass, suggest skipping tests due size/time.
- Run server/client from module folders via exec plugin:
  - Mage.Server: `mvn -q exec:java`
  - Mage.Client: `mvn -q exec:java`
- Server reads config relative to working dir (`config/config.xml`) unless overridden with `-Dxmage.config.path=...`.
